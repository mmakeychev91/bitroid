# AGENTS.md — руководство для AI-агентов

Файл с правилами и контекстом проекта для ассистентов (Cursor, Copilot и др.).

## Стек и сборка

- **Сборка:** Gulp
- **Исходники:** `src/` (HTML, SCSS, JS)
- **Результат:** `app/` (собранные HTML, CSS, JS, шрифты, изображения)
- **Команды:** `yarn dev` / `npm run dev` — разработка; `yarn build` / `npm run build` — продакшен

## Структура

- `src/` — исходный код
  - `*.html`, `index.html`, — страницы
  - `partials/` — включаемые шаблоны (хедер, футер, каталог, фильтры и т.д.)
  - `js/` — модули: `main.js`, `components/`, `utils/`
  - `scss/` — стили: `main.scss`, `config/` (vars, mixins, global), `components/`, `mixins/`, `ui/`
- `app/` — результат сборки (не редактировать вручную)

## Стили (SCSS)

### БЭМ

- **Блок:** `block` — независимый компонент
- **Элемент:** `block__element` — часть блока
- **Модификатор:** `block--modifier` или `block__element--modifier` — вариация
- **Один БЭМ-блок — один файл:** каждый блок выносится в отдельный `_block-name.scss`; крупные разделы (например, главная) собираются в папку с `_index.scss`

### Брейкпоинты (mixins/breakpoint)

| Миксин            | Диапазон          |
|-------------------|-------------------|
| `@include tablet-plus` | min-width: 1280px (десктоп) |
| `@include tablet` | max-width: 1279px (планшет и меньше) |
| `@include only-tablet` | 768px – 1279px (только планшет) |
| `@include mobile` | max-width: 767px (мобилка) |
| `@include mobile-plus` | min-width: 768px |
| `@include desktop-narrow` | 1280px – 1400px |

### Правила для брейкпоинтов

- **Писать внутри селектора** — не выносить `@include tablet` / `@include mobile` в отдельные блоки в конце файла
- **Селекторы не повторять** — один селектор = одно место, все варианты экранов внутри него

  ```scss
  .product {
    display: flex;

    @include tablet {
      display: grid;
    }

    @include mobile {
      display: block;
    }
  }
  ```
- **Лайфхак при сильных отличиях десктоп/мобилка:** если стили на десктопе и мобилке сильно различаются, можно использовать `utils/_breakpoint-condition.scss` и **дублировать разметку** — отдельные блоки с классами `.desktop`, `.only-tablet`, `.tablet`, `.mobile`, которые скрываются на «чужих» экранах. Так каждый вариант получает свои стили без сложных переопределений.
- **Переменные:** цвет, типографика, отступы — в `scss/config/_vars.scss` и глобальные в `_global.scss`. При использовании переменных в стилях указывать **и переменную, и запасной цвет** (fallback), например: `background: var(--second-grey-300, #D6D6D6);` — не только `var(--second-grey-300)`.
- **Обычный текст по умолчанию:** десктоп — `font-size: 16px; font-style: normal; line-height: normal;`; мобилка (`@include mobile`) — `font-size: 14px;`. Использовать для основного текста, если в макете не указано иное.

## JavaScript

- ES-модули: `import`/`export`.
- Компоненты подключаются в `src/js/main.js` и инициализируются при `DOMContentLoaded`.
- Не добавлять глобальные переменные без необходимости; предпочтительно инкапсуляция в модулях.

## Язык и комментарии

- Ответы пользователю и комментарии в коде — на **русском**, если в запросе не указано иное.
- В коде комментарии на русском допустимы и приветствуются для пояснения логики/верстки.

## Верстка по макету (pixel-perfect)

- Агент выступает как **senior frontend pixel-perfect верстальщик**. Не придумывать ничего от себя: размеры, цвета, отступы, шрифты копировать **1:1 из Figma**.
- Для верстки по макету использовать **get_design_context** (Figma MCP): вызывать его для выделенного пользователем узла/фрейма в Figma и получать от Figma MCP код/данные по выделенному узлу — **оттуда** копировать размеры, цвета, отступы, шрифты в разметку и стили. Так можно максимально не придумывать от себя.

## API-ключи

- **Яндекс.Карты (страница Контакты):** ключ разработчика API Яндекс.Карт подставляется в атрибут `data-yandex-maps-api-key` у контейнера карты `#contacts-map` в `src/partials/contacts/contacts-content.html`. Без ключа на месте карты показывается подсказка. Ключ можно получить в [кабинете разработчика Яндекс.Карт](https://developer.tech.yandex.ru/).

## Рекомендации при изменениях

1. Редактировать файлы в `src/`, не в `app/`.
2. Проверять использование брейкпоинтов и не ломать адаптив (планшет/мобилка).
3. Сохранять единообразие: БЭМ, принятые в проекте отступы и шрифты.
5. **При добавлении новой страницы** (новый `*.html` в `src/`) — добавить строку в навигационный список в `src/index.html`: новый пункт `<li><a href="имя-страницы.html">Подпись ссылки</a></li>` в `<ul>` внутри `main`, чтобы страница была доступна с главной.
